## 1. 概要

本システムは、**レース一覧・詳細表示、動画確認、解析結果補正、レビュー、確定、差し戻し、再解析、突合、CSV出力、処理管理、解析ツール管理**を行うための API 群で構成する。

フロントエンドは React、バックエンドは FastAPI を前提とし、認証は GCP IAP、アプリ実行は Cloud Run、通常解析は Cloud Scheduler 起点、重い解析処理は Cloud Batch / GPU VM、業務DBは Cloud SQL を利用する。再解析は Cloud Run 側からジョブ実行を起動する構成とする。

API は大きく次の単位に分ける。

- 一覧・詳細表示
- 補正フロー
- レビュー / 確定 / 差し戻し
- 同期 / CSV / 一括変更
- 解析 / 突合
- 処理管理
- 解析ツール管理

## 2. 時刻仕様の固定

### 2-1. 基本方針

- API で返す**日時はすべて ISO 8601 形式**
- タイムゾーンは **JST（`+09:00`）固定**
- バックエンド内部が UTC 管理でも、API入出力は JST 基準とする

### 2-2. `date`

- `GET /races` や CSV 出力条件の `date` は、**JST 基準の日付**
- 日付の自動補完も JST 基準で行う

### 2-3. `updatedAt`

- 一覧・詳細で返す `updatedAt` は **JST の日時**
- `confirmedAt` / `completedAt` / `savedAt` / `requestedAt` なども同様に JST で返す

### 2-4. 目的

Cloud Run、DB、バッチ実行環境の時差や日付またぎによるズレを防ぐため。特に `GET /races` の初期日付補完や一覧の `updatedAt` は JST 固定を前提にする。

## 3. 認証方式

### 3-1. 認証

- **GCP IAP 認証必須**
- API は IAP 経由で受け取り、アプリ側で利用者を識別する

### 3-2. 認可

**一般ユーザー**

- 一覧 / 詳細閲覧
- 補正開始 / 一時保存 / 完了 / キャンセル
- 再解析要請
- 突合申請

**管理者**（上記に加えて）

- 確定 / 差し戻し
- 再解析実行
- 突合再実行
- 強制ロック解除
- 一括ステータス変更
- 同期 / CSV出力 / 処理管理 / 解析ツール管理

---

## 4. 利用制限

- `GET /races` はページングなしを基本とし、最大想定件数は約 36 件
- `GET /races/{raceId}` は軽量集約 API とし、重いデータは別 API に分離する
- 同一レースは **1つの ACTIVE な補正セッションのみ**
- heartbeat は編集中のみ約 1 分おき送信を想定
- heartbeat が 30 分以上途絶えたセッションはタイムアウト対象とする
- CSV 出力は非同期ジョブ方式とする

---

## 5. セキュリティ

- 動画再生 URL と CSV ダウンロード URL は**署名付き URL**
- GCS の内部パスは通常レスポンスに露出しない
- 他人が保持している編集中セッションには保存 / 完了 / heartbeat 不可
- 強制ロック解除は管理者のみ
- バッチジョブ削除は論理削除を前提
- 高頻度な heartbeat は通常 `audit_log` に残さない
- 履歴一覧と監査ログ画面は役割を分離する

## 7. 権限差分の明記

### 7-1. `GET /races/{raceId}` の `availableActions`

**常駐（全ロール共通）**

- `OPEN_ANALYSIS_OPTION`
- `OPEN_HISTORY_COMMENT`

**一般ユーザー**

| 状態 | availableActions |
|------|-----------------|
| `ANALYZED + null` | `START_CORRECTION`, `REQUEST_REANALYSIS`, `REQUEST_LINKAGE` |
| `ANALYZED + REVISION_REQUESTED` | `START_CORRECTION`, `REQUEST_REANALYSIS`, `REQUEST_LINKAGE` |
| `ANALYZED + EDITING` かつ本人ロック中 | `SAVE_DRAFT`, `COMPLETE_CORRECTION`, `CANCEL_CORRECTION` |

| `NEEDS_ATTENTION + MATCH_FAILED` | `REQUEST_LINKAGE` |
| `NEEDS_ATTENTION + ANALYSIS_FAILED` | `REQUEST_REANALYSIS` |

| 状態 | availableActions |
|------|-----------------|
